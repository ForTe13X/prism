# Prism — 旁观者观察笔记 (Observer Notes)

> 这是一份**外部旁观者**的读法,刻意带点距离感——写它的视角不泡在日常搭建里,所以能说一些
> "当局者容易自我说服掉"的话。**它是一个视角,不是路线令**:take it or leave it,最终由建造者
> 拍板。本笔记只谈项目分析,不含任何外部/专有信息。

---

## 1. Prism 现在到底是什么(诚实定性)

一个**为了好玩与学习的实验场**(joy-of-learning sandbox),把几个想法揉在一起测试:
- 语义地基驱动的领域无关驾驶舱(UI = f(spec));
- 时间帧 + 回放;
- 本体图谱(版本回放);
- 轨迹仿真(决策支持方向);
- (候选)LLM 生成 spec。

**它不是产品,也不是简历作品**——这是作者**主动做的决定**,不是缺陷。因此它的成败标准只有两条:

1. **还有意思吗?** 一旦变成义务/负担,就停或换。
2. **还在学新东西吗?** 只是在堆早会写的 CRUD,才是这个框架下唯一的"垃圾":无聊。

这两条之外的任何尺子(PMF、护城河、招聘信号),在当前定位下**都不适用**,不要用它们评判这个项目。

## 2. 价值定位的推演(留档,便于理解"为什么落在 for fun")

作者依次审视过三种定位,逐层剥掉自己其实不在乎的:

- **产品向(已搁置)** —— 难点是真实的:① 没人买"领域无关驾驶舱",买家要的是某个具体痛点决策;
  ② 决策仿真的**可信度/信任**是整个成败点(看着权威实则 wiggle 会反噬);③ 通用仪表盘是红海
  (Grafana/Retool/Metabase);④ "做了又没什么损失"是错觉——造价降了,但**注意力/信任/维护**成本没降。
  > 一句话:作为产品要先**垂直化 + 验证买家与信任**,再谈引擎;这是另一个量级的承诺。
- **简历向(已搁置)** —— 评判标准变成"30–90 秒内让资深工程师想约你聊";关键是**深度 + 打磨 + 可读 > 广度**
  (铺一堆半成品反而是减分)。
- **for fun 向(当前生效)** —— 含义:**跟着好奇心走、hodgepodge 是正解、不背连贯性税、可随时弃坑、
  优先选学习密度高的线**。

> 留这段不是要翻案,是把推理留档——别把"产品/简历"的那套要求误当成当前的指令。

## 3. 一个耐放的内核(仅当定位改变时才相关)

如果哪天它真要超越玩具,最站得住的内核是:**语义优先的决策支持 / 轨迹仿真**——把"领域的语义模型 →
自动长出带演化/回放/what-if/归因的决策 cockpit"做出来,而**诚实的置信度**是差异点(满市场都是自信
胡说的 AI,会明说"我有多不确定"的工具才让人敢按下去)。这条记下来以免丢失,但**标注为条件性**:只在
定位升级时才捡起。

---

## 4. 反"工业垃圾"判据(按 for-fun 重述)

"垃圾"只相对于"本该有用却没用"才成立;一个为学习而搭的东西,哪怕只有作者一个人用、哪怕半途弃坑,也
已兑现全部目的。所以这个框架下唯一要警惕的两种"垃圾":

- **义务式铺摊子** —— 因为"反正能造"就把功能堆全,做着做着变成 chore。解药:只做现在最挠你的那块。
- **零学习增量** —— 重复你早就会的活。解药:优先学习密度高的方向(见 §6)。

---

## 5. 诚实纪律(第一公民,尤其对仿真)

无论玩不玩,这条不降级:

- 合成/外推的数值**永远标注**,不冒充实测或权威;
- 仿真**永远带不确定带**,且标清"真实数据锚定 / 示意外推 / 声明动力学"三类来源;
- `identifier` 之类不演化的量逐帧字节一致(回放一致性的命门);确定性(无 random/clock)。

理由:决策支持类特性一旦"看着权威实则幼稚",其下行不被造价封顶——会反噬整个东西的可信。

---

## 6. 旁观者对三条线的读法(按学习密度,不是按"对错")

- **轨迹仿真** —— 系统思维 + 数学 + 可视化汁水最足:通用动力学怎么建、蒙特卡洛 vs 声明式规则、
  不确定性如何传播;而且拖滑块滑进"模拟出来的未来"本身就最出片。
- **LLM 生成 spec** —— 学 structured-output / agent 编排的好载体;成的那刻最"魔法"。
- **本体图谱画布** —— 力导向布局 / canvas 渲染 / 版本回放,图形与算法练手。

没有谁更"对"。**哪条现在最挠你,就先抓哪条。**

## 7. 旁观者已放进仓库的东西(隔离、未接线)

为不干扰在建会话,以**新文件**加了一个**独立的轨迹仿真引擎**(决策支持方向的最小内核):

- `backend/app/simulation.py` —— `simulate(spec, entity, attribute, horizon, scenarios, row_index, baseline)`
  → baseline + N 情景轨迹,每条含不确定带(9 次确定性 roll)、越限帧、`verdict`(优先避免越限,否则
  最低终值)。动力学在 spec 内声明(`dynamics`: mean-revert `rate`/`trend`/`volatility`,均可缺省);
  情景 `shift`(设定点变更)/`pulse`(一次性)。自包含确定性 hash,**不 import `data_synth`**,只读
  `specs_loader.load_spec`。
- `backend/app/sim_routes.py` —— 独立 `POST /api/sim/{spec_id}`,**未**接入 `main.py`。
- `backend/tests/test_simulation.py` —— 10 项,本地 app(不 import `main`),全过。

**接线点(择时,引擎无需改):**
1. `main.py` 一行:`from .sim_routes import sim_router; app.include_router(sim_router)`。
2. 真实状态:`simulate(..., baseline=<当前帧实测值>)` 即从 live 投影(现为自包含合成基线)。
3. 前端:某 tab 调 `POST /api/sim/{spec}`,把 `trajectories[].frames{lo,mid,hi}` 画成 slider `now`
   右侧的扇形带 + 情景对比 + `verdict`,并按 §5 诚实标注 `confidence`。

待留意 / 整改:见 [`DEMANDS.md`](DEMANDS.md) §3(不重复)。最高优先级一句:**已建好的 `/api/sim` 仿真引擎仍未接线(main + 前端),最差异化的那块在睡觉**。

## 10. 停放的远期好奇(parked,非现在做)
- **超图 / nexus 实体 / 高阶表示**(MIT 一篇,科学发现向):成对关系图 → **n-元超边**(高阶关系),用**跨学科桥接的 nexus 实体**连多域,**遍历**超图 surface 跨域类比 → 引导发现。
  - **与现状的接续(不是岔路)**:Prism 的本体 + canonical 实体解析 + 刚设计的跨源 link([`DESIGN_data_package`](DESIGN_data_package.md) §4d)正是它的**地基**;成对关系(P2)→ 超边、单域 → 跨域 nexus(Prism 本就多 spec)、答查询 → **遍历发现**(接自演化 / 归因线)。
  - **诚实的硬处**:遍历**永远**能 surface "联系";难的是**判定哪条是真洞见 vs 伪相关**(与 LLM-胡说同一家),价值与难度全在**验证 / 排序**,不在建图。"real discovery" 是 moonshot;但表示 / 遍历 / 类比框架对 for-fun 学习汁水足。
  - **为什么现在停放是对的**:让它非垃圾的前提(实体解析 / 跨源链 / 诚实验证)正是当前数据基底 / axiom 工作在打的地基——**现做的就是它的前置**,不是绕路。未来 revisit。
